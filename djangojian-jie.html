django簡介

Date 週三 08 一月 2014 Tags django

what's new in django 1.6

fat model, thin view, stupid template

每一個 app 應該專注在自己簡單的事務上

過於複雜則需分離成多個 app

setting 的部分 最好分成

  1. base 基礎構成再 base完成
  2. local local dev
  3. staging
  4. test
  5. product

完整 product 使用

如有用 git 來做版本控制

最好把 一些重要的東西 脫離這個

如 secretkey api key 這些東西會造成安全上的一些問題

啟動 server with setting

django-admin.py runserver --settings=root.settings.local

除了 setting 還有 requirements

應該具有的是

_base.txt(最基礎需要的套件)

依據不同階段有不同的 需要 package

local.txt

staging.txt

production.txt

django 開發時 應該具備 兩種以上的 database engine

因為不同的 database 對於 field 的反應不一

只用一種開發是可以的

但是 假如你開發時用的跟 正式發行 (product ) 是不同一個 database engine

這就需要考量 sqlite3 是 若型態 weak type field

各個 database 其反映個不同 對於在資料進行轉移時 可能會遇上在開發時沒遇到的事情

現在我只有用 sqlite3 現在要考慮 一下 postgreSQL

我不想用 Mysql 勒 $$帝國底下的產物
